# ADR 0002: transferencia inicial y firma asimétrica

Estado: aceptado como dirección.

La primera forma de pago es transferencia con verificación manual. El diseño mantiene el proveedor de pago reemplazable. Las licencias futuras se firmarán asimétricamente; la clave privada permanece en infraestructura controlada y la aplicación instalada valida con una clave pública.
